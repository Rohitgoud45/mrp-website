Executive Summary

The City of Riverton faces critical operational challenges within its municipal parking infrastructure. Current workflows are fragmented across the Parking Permit Office, Enforcement Division, and Finance Department, relying on outdated batch-processing, legacy hardware, and siloed data repositories. This lack of integration leads to severe synchronization delays, resulting in wrongful citations, revenue loss, and diminished public trust.

To address this, Team 7 proposes the implementation of the Centralized Integrated Parking System (CIPS). CIPS is a cloud-native platform utilizing a robust Node.js/Express backend and a centralized SQL database to ensure real-time data synchronization across all municipal departments. By transitioning from batch updates to instant verification, CIPS enables enforcement officers to access live permit statuses, significantly reducing erroneous tickets.

This project encompasses a complete enterprise architecture redesign following the TOGAF ADM framework. It includes the development of citizen-facing portals, a mobile app for enforcement officers, and a comprehensive Looker Studio analytics dashboard for city administrators. Ultimately, CIPS aims to increase operational efficiency by 40%, reduce wrongful citations by over 80%, and establish a scalable foundation for Riverton's smart city initiatives.

Problem Context & Stakeholders

The Core Problem

Riverton's current parking management environment exemplifies severe misalignment across Business, Data, and Technology layers. Specifically, the system suffers from:

  • Delayed batch-based data synchronization (24-48 hour lag).
  • Disconnected legacy handheld devices used by enforcement.
  • High volume of wrongful citations due to stale permit data.
  • Frustrating, manual dispute resolution processes for citizens.

These inefficiencies drain municipal resources and damage citizen satisfaction.

Data Flow Explanation

How data moves through the new ecosystem in real-time:

  1. Citizen Portal: User submits permit request & payment.
  2. Cloud DB: Request is instantly written to the centralized SQL database.
  3. Admin Engine: Automated checks approve routine permits.
  4. Enforcement API: Officer's mobile app queries DB via REST API upon plate scan.
  5. Live Response: App displays exact, up-to-the-second permit status, preventing wrongful tickets.

Proposed Solution & Architecture

Enterprise Architecture Overview

The Centralized Integrated Parking System (CIPS) utilizes a modern, 4-tier cloud-native architecture mapped via TOGAF ADM. It replaces localized, siloed servers with a unified infrastructure.

Solution App Showcase

The CIPS Enforcement & Citizen App

The application acts as the primary presentation layer for the CIPS ecosystem. It features role-based access control (RBAC):

  • For Citizens: A responsive web app to apply for permits, upload verification documents, and securely pay fines.
  • For Officers: A high-contrast mobile PWA (Progressive Web App) allowing instant license plate lookups via the live API, eliminating the "stale data" issue.

By connecting directly to the central SQL database, the app addresses the main challenge: Data Fragmentation.

Projected Improvements on KPIs

Implementing CIPS will drive the Wrongful Citation Rate down from 45.5% to 5.25%, while drastically reducing Permit Latency from 168.2 hours to 8.61 hours. Dispute Resolution Time is projected to fall from 231.3 hours to 115.3 hours due to automated checks.

Ethical & Social Considerations

Digitizing municipal enforcement requires careful ethical boundaries:

  • Data Privacy (PII): All citizen data (names, addresses, plates) is encrypted at rest and in transit. Access is strictly audited.
  • Surveillance Concerns: The system is designed for compliance checking, not tracking. Location data is only logged at the moment of a citation.
  • Digital Equity: Acknowledging that not all citizens have smartphone access, physical permit application centers will remain operational and sync to the new cloud system.

Lessons Learned & Team Reflection

Through this project, Team 7 gained critical insights into enterprise architecture and public sector digital transformation:

  • Technical: We learned the importance of API-first design. Transitioning from batch to real-time sync is complex but absolutely necessary for modern systems.
  • Business: Stakeholder alignment was harder than expected. Balancing the needs of citizens (privacy) with enforcement (accuracy) required continuous compromise.
  • Teamwork: Utilizing Git for version control and Agile methodologies kept our cross-functional team on track during the tight 12-week development window.
